Scope

Javascript has function scope & global scope. If you were following instructions from two kit-sets to construct a shelf & coffee table & the instructions clearly labeled the utility screws & where they were to go you would have no problem. Errors or mistakes would occur if you were to take a coffee table screw ( unbeknown to you, of quite a different size, capability & unlabeled ) & affix this screw in to the shelving unit - well it is not going to work. Unless this particular screw is of universal use which in javascript means globally accessible by any other code that wants to use it - then by all means a universal screw is perfect for the job but it would have to be a very special screw. The distinctly labeled screw is the to javascript the function scope & can only be used with the coffee table or shelf - not both.

Parentheses ()

If the build coffee table instructions above where defined in a javascript function it would begin with something like this - function buildTable().... The () must match & can or can not have parameters. In this example we could have three - like function buildTable(item, x,number).... The item, x & number are inputs that the following block of code will work with & execute. If you then want to use the function you call it like this - buildTable(screw, 30,20).... The screw is the item, 30 is screw locations & the 20 is the number of screws. These are arguments that the code manipulates & executes on.

Brackets [ ]

Mis-matched brackets in javascript causes errors. If your kit set instructions missed a valuable word or finalizing step -could be your coffee table becomes a boggy board. On your completed shelf(?) you have a collection of valuable & rare books divided into categories with shelf partitions. One partition does not appear to be in its right location or there at all - your valuable & rare books take a tumble.

Braces { }

Following the buildTable(item, x, number)... we add { }. Inside these braces is the instruction block which acts on the item, x, number & any other declared variables in the function. Errors are made when there are not matching opening & closing braces. For this example could be something like this - { if ( number === x ) do something ...}.

Single Quotes ' ' & Double Quotes " "

Javascript excepts both single & double quotes when declaring or outputting a string value - which is seen as text & based on how you want the text to be treated & behave. Any character between the quotes whether it is a numeral, letter & so on is treated as text. Consistency requires using either ' ' or " " throughout the code. However if you wanted to claim emphasis on a word/s in an output message from your function {...var message = "Coffee table completed 'Hurray!' "; .... } - you just make sure to use the alternative quote style so Javascript does not get confused.
